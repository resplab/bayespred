Pragmatic Bayesian Prediction Modelling with bayespred

Overview

bayespred implements the pragmatic Bayesian workflow for clinical prediction modelling described in Sadatsafavi & Riley (2026). The key insight is that the posterior mean (PM) predicted probability — rather than the plug-in estimate (PE) — is the correct target at deployment time under a proper scoring rule.

This vignette illustrates the full workflow on a simulated dataset that mimics a binary 30-day outcome study.

Simulated dataset

library(bayespred)
set.seed(2026)
n <- 2000

dat <- data.frame(
  age    = rnorm(n, mean = 65, sd = 12),
  sbp    = rnorm(n, mean = 130, sd = 20),
  sex    = factor(sample(c("male", "female"), n, replace = TRUE)),
  killip = factor(sample(c("I", "II", "III"), n, replace = TRUE,
                         prob = c(0.7, 0.2, 0.1)))
)
lp <- -4 + 0.04 * dat$age - 0.02 * dat$sbp +
       0.3 * (dat$sex == "male") +
       0.8 * (dat$killip == "II") + 1.5 * (dat$killip == "III")
dat$outcome <- rbinom(n, 1L, plogis(lp))
cat(sprintf("n = %d,  events = %d (%.1f%%)\n",
            n, sum(dat$outcome), 100 * mean(dat$outcome)))
#> n = 2000,  events = 78 (3.9%)

Fitting under different priors

form <- outcome ~ age + sbp + sex + killip

fit_flat  <- bpmfit(form, data = dat, prior = flat())
fit_jeff  <- bpmfit(form, data = dat, prior = jeffreys())
fit_logf  <- bpmfit(form, data = dat, prior = log_f(m = 2))
fit_ridge <- bpmfit(form, data = dat, prior = bridge())

Compare coefficient estimates across priors:

coef_tab <- sapply(
  list(flat = fit_flat, jeffreys = fit_jeff, logf2 = fit_logf, ridge = fit_ridge),
  coef
)
round(coef_tab, 3)
#>               flat jeffreys  logf2  ridge
#> (Intercept) -3.954   -3.919 -3.922 -3.687

Prediction for a new patient

sam <- data.frame(age = 72, sbp = 105, sex = "male", killip = "II")

Plug-in estimate vs posterior mean

pe <- predict(fit_logf, sam, method = "pe")
pm <- predict(fit_logf, sam, method = "pm")
cat(sprintf("PE = %.3f   PM = %.3f\n", pe, pm))
#> PE = 0.063   PM = 0.066

Credible intervals

predict(fit_logf, sam, method = "pm", interval = 0.95)
#>          fit       lwr      upr  se.link
#> 1 0.06556389 0.0368101 0.106911 0.291278
predict(fit_logf, sam, method = "pe", interval = 0.95)
#>          fit       lwr      upr  se.link
#> 1 0.06335301 0.0368101 0.106911 0.291278

The PM lies inside but not at the centre of the interval (Jensen’s inequality).

MacKay approximation

predict(fit_logf, sam, method = "pm_mackay")
#> [1] 0.06600115

PM projection for simplified deployment equations

bpmproject() with type=‘pm’ regresses the posterior-mean soft labels back onto a design matrix, producing a standalone bpmproj_pm object with no covariance matrix. It can use the same predictors as the main model (self-projection) or a simpler custom formula.

proj <- bpmproject(fit_logf, type="pm")
round(coef(proj), 3)
#> (Intercept)         age         sbp     sexmale    killipII   killipIII 
#>      -3.886       0.017      -0.006       0.322       0.374       1.150

predict(proj, sam)
#> [1] 0.06579206

The projected model is self-contained and can be saved and reloaded independently of the original fit.

All four priors on the same patient

sapply(
  list(flat = fit_flat, jeffreys = fit_jeff, logf2 = fit_logf, ridge = fit_ridge),
  predict,
  newdata = sam,
  method = "pm"
)
